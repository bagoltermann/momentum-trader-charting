### Streaming Seed Fix

**File:** `src/renderer/store/candleDataStore.ts`
**Commit:** `2807759`

When Schwab returns all-zero placeholder candles and streaming quotes eventually arrive, `updateStreamCandle` now seeds the candle array instead of returning early on `primaryCandles.length === 0`. Clears "No trades yet" error.

### Streaming Health Monitor

**Backend:** `backend/api/routes.py` — New `GET /api/streaming/health` proxy to trader app's watchdog endpoint (same httpx pattern as rotation proxy).

**Hook:** `src/renderer/hooks/useStreamingHealth.ts` (NEW)
- Feature flag: `STREAMING_HEALTH_ENABLED = true`
- 10s polling, AbortController + mountedRef cleanup
- Derives status from `last_message_seconds_ago`: <5s=live, 5-30s=delayed, >30s=stale
- Returns formatted quote count (42K, 1.2M)

**StatusBar:** `src/renderer/components/layout/StatusBar.tsx`
- Replaced "Data: Streaming/Polling" with health-aware indicator
- `Feed: LIVE (42K)` (green), `Feed: DELAYED 15s` (yellow), `Feed: STALE 45s!` (red)
- Falls back to old behavior when health endpoint unavailable

**App.tsx:** Imported `useStreamingHealth`, passed to `StatusBar`.
